Lecture 4: The Science of Prompting: Chain-of-Thought & Few-Shot


Introduction: Why Prompting is a Skill
In 2026, "prompt engineering" has evolved from a curiosity to a core professional skill. The difference between a novice and an expert prompt engineer can be:


10x better outputs - More accurate, relevant, and useful


50% cost reduction - Fewer API retries and iterations


Professional credibility - Shows you understand how LLMs work
This lecture transforms you from a casual ChatGPT user to a power user who understands the science behind effective prompts.


The Four Pillars of Effective Prompting
Every great prompt contains four elements:


1. Persona (WHO is responding)
Defines the role and expertise level of the AI.
Weak example:

Tell me about machine learning.
Strong example:

You are a senior ML engineer at a fintech company. Explain machine learning to a business stakeholder who needs to understand ROI but has no technical background.
Why It Works:


Sets the knowledge level (expert, not encyclopedia)


Defines the audience (business, not technical)


Establishes context (fintech, not generic)
Persona Templates:
For technical explanations:

You are an expert [role] with 10 years of experience in [domain].
For creative content:

You are a [profession] known for [specific style/trait].
For business analysis:

You are a [senior role] at a [company type] analyzing [specific challenge].


2. Task (WHAT you want done)
The specific action you want the AI to perform.
Weak Tasks:


"Tell me about…" (too vague)


"Explain…" (what aspect? how deep?)


"Write something…" (what exactly?)
Strong Tasks:


"Generate 5 SQL queries that…"


"Summarize this article in 3 bullet points focusing on…"


"Debug this Python code and explain what was wrong"
Task Verb Library:
Goal
Strong Verbs
Analysis
Analyze, Compare, Evaluate, Assess, Critique
Creation
Generate, Create, Design, Draft, Compose
Transformation
Convert, Translate, Reformat, Restructure
Extraction
Extract, Identify, List, Enumerate, Find
Explanation
Explain, Clarify, Illustrate, Demonstrate


3. Context (BACKGROUND information)
Provides the AI with necessary background to give relevant answers.
Context Types:
Domain Context:

I work in a healthcare startup building patient monitoring systems.We're evaluating ML frameworks for real-time anomaly detection.
Constraint Context:

Budget: ₹50,000/month for cloud servicesTeam size: 3 junior engineersDeployment: Must work on-premise due to HIPAA compliance
Audience Context:

This report will be presented to:- CFO (cares about cost and ROI)- CTO (cares about technical feasibility)- Legal team (cares about compliance)
Example with Full Context:

CONTEXT:- Company: B2B SaaS selling to mid-market companies- Current: 500 support tickets/day, 12-hour response time- Goal: Reduce to 2-hour response time- Constraint: Can't hire more support staffTASK:Design a 3-phase implementation plan for an AI-powered customer support system that integrates with our existing Zendesk setup.For each phase, specify:- Technical components- Estimated cost- Time to implement- Expected impact on response time


4. Format (HOW you want the output)
Specifies the structure and style of the response.
Format Options:


Table


Bullet points (with specific number)


Numbered list


JSON object


Markdown document


Python dictionary
For Code:

Provide the solution as:1. Complete, runnable Python code2. Inline comments explaining each section3. Example usage at the end4. Requirements.txt file contents
For Analysis:

Structure your analysis as:## Executive Summary (2-3 sentences)## Key Findings1. [Finding with supporting data]## Recommendations- [Actionable recommendation with timeline]## Risks and Mitigation- [Risk]: [Mitigation strategy]


Advanced Technique 1: Chain-of-Thought (CoT) Prompting


What is Chain-of-Thought?
Instead of asking for just the answer, you ask the AI to show its reasoning process.
Why It Works:


LLMs are trained on text that includes reasoning (tutorials, explanations)


Forcing step-by-step thinking reduces errors


Makes debugging easier (you can see WHERE it went wrong)


Basic Chain-of-Thought
Method 1: Explicit Instruction
Before (Direct Answer):

Calculate the ROI of implementing an AI chatbot if:- Implementation cost: ₹10 lakhs- Reduced support staff cost: ₹3 lakhs/year- Increased customer satisfaction: Estimated ₹2 lakhs/year value
After (Chain-of-Thought):

Calculate the ROI of implementing an AI chatbot.Think step-by-step:1. First, identify all costs2. Then, identify all benefits3. Calculate yearly ROI4. Determine payback period5. Provide final recommendationGiven:- Implementation cost: ₹10 lakhs- Reduced support staff cost: ₹3 lakhs/year- Increased customer satisfaction: Estimated ₹2 lakhs/year value
Example Output:

Step 1: Identify all costs- One-time implementation: ₹10,00,000- Ongoing maintenance (estimated): ₹50,000/yearStep 2: Identify all benefits- Support staff cost reduction: ₹3,00,000/year- Customer satisfaction value: ₹2,00,000/year- Total annual benefit: ₹5,00,000/yearStep 3: Calculate yearly ROIROI = (Annual Benefit - Annual Cost) / Investment × 100ROI = (₹5,00,000 - ₹50,000) / ₹10,00,000 × 100ROI = 45% per yearStep 4: Payback periodPayback = Investment / Annual Net BenefitPayback = ₹10,00,000 / ₹4,50,000 = 2.22 yearsStep 5: RecommendationStrong business case. 45% ROI with 2.2-year payback is excellent for enterprise software. Recommend proceeding with implementation.

Advanced Technique 2: Few-Shot Learning


What is Few-Shot Learning?
Teaching the AI by providing examples of input-output pairs.
When to Use:


Consistent formatting needed


Domain-specific transformations


Custom classification tasks


Style matching


Anatomy of a Few-Shot Prompt

I need you to classify customer support tickets into categories.Here are examples:Input: "I can't log into my account, forgot my password"Output: CATEGORY: Authentication | PRIORITY: Medium | DEPARTMENT: IT SupportInput: "Charged twice for the same order #12345"Output: CATEGORY: Billing | PRIORITY: High | DEPARTMENT: FinanceInput: "How do I export data to CSV format?"Output: CATEGORY: Feature Question | PRIORITY: Low | DEPARTMENT: Product SupportNow classify this ticket:Input: "My credit card was declined but I have sufficient funds"Output:
Expected Response:

CATEGORY: Payment Processing | PRIORITY: High | DEPARTMENT: Finance


Few-Shot Best Practices
1. Diverse Examples
Include edge cases and variations:


Example 1: Simple positive case


Example 2: Simple negative case


Example 3: Ambiguous/edge case


Example 4: Complex case with multiple elements
2. Consistent Format
All examples must follow the EXACT same format. Use the same structure (e.g., "Input:" and "Output:") throughout.
3. Progressive Complexity
Start simple, build to complex:

Example 1: "Meeting at 3pm" → {"time": "15:00", "date": "today"}Example 2: "Coffee next Tuesday at 10am" → {"time": "10:00", "date": "2026-04-29"}Example 3: "Quarterly review first Monday of next month 2pm" →            {"time": "14:00", "date": "2026-05-05", "recurring": "quarterly"}

Prompting Anti-Patterns (What NOT to Do)


Anti-Pattern 1: The Vague Ask
Bad: Tell me about AI
Good: Explain how transformer models work, focusing on the attention mechanism, with examples from GPT architecture. Target audience: software engineers new to NLP.


Anti-Pattern 2: The Kitchen Sink
Bad: Analyze this data, create visualizations, write a report, suggest improvements, and also build a dashboard
Good: Break into 5 separate, focused prompts


Anti-Pattern 3: Assuming Context
Bad: How should I improve it? (AI doesn't know what "it" refers to)
Good: How should I improve the response time of my REST API that currently takes 800ms for user authentication requests?


Anti-Pattern 4: No Constraints
Bad: Write code for a chatbot
Good: Write Python code for a chatbot using Flask, OpenAI API, and SQLite. Target deployment: Heroku free tier. Must handle 100 concurrent users.


Anti-Pattern 5: Forgetting to Validate
Bad: Accept AI output as truth without verification
Good: "Provide sources for each claim" or "Mark any uncertain statements"

Lecture 6: Hallucinations & Bias: When AI Fails and Why


Introduction: The Dark Side of AI
AI systems in 2026 are incredibly powerful, but they have critical flaws:


Hallucinations: Confidently stating false information


Bias: Reflecting and amplifying societal prejudices


Brittleness: Failing in unexpected ways
Understanding these failures isn't academic - it's essential for deploying AI responsibly in production.


Part 1: Hallucinations - The "Stochastic Parrot" Problem


What Are Hallucinations?
Definition: When an AI generates plausible-sounding but factually incorrect information.
Real Example:

User: "Who won the Nobel Prize in Physics in 2025?"AI: "Dr. Sarah Chen from MIT won for her work on quantum entanglement."Reality: This person doesn't exist. The AI invented a plausible-sounding answer.


Why Do LLMs Hallucinate?
Reason 1: Training on Patterns, Not Facts
LLMs learn statistical patterns in text, not truth.
Reason 2: Optimized for Plausibility, Not Accuracy
LLMs are trained to predict the next word that SOUNDS right, not the word that IS right.
Reason 3: Knowledge Cutoff
Even when the AI "knew" something, information changes over time.
Reason 4: Pressure to Always Respond
LLMs are trained to NEVER say "I don't know" (because that's rare in training data).


Types of Hallucinations
1. Factual Hallucinations (Invented Facts)

User: "Summarize the research paper 'Attention is All You Need'"AI: "The paper was published in 2015 by researchers at MIT..."Reality: Published in 2017 by Google Brain (not MIT)
2. Contradictory Hallucinations (Inconsistency)
Contains self-contradicting information within the same response.
3. Fabricated Source Hallucinations

User: "Cite sources for the claim that AI reduces productivity"AI: "According to a 2025 study by McKinsey & Company, 'AI Implementation and Productivity: A Longitudinal Analysis', companies saw 23% productivity decline in the first year..."Reality: This study does not exist.
4. Instruction Hallucinations (Following Non-Existent Instructions)
AI claims to have capabilities it doesn't have (e.g., database access).


Detecting Hallucinations
Technique 1: Ask for Sources
If the AI cites papers that don't exist (search for them!), it's hallucinating.
Technique 2: Ask the Same Question Multiple Ways
If answers contradict each other, at least one is hallucinated.
Technique 3: Request Confidence Levels

Answer this question and rate your confidence (0-100%) for each claim:What are the pricing tiers for Pinecone vector database?
Technique 4: Verify Critical Information
For Production Systems - Never trust AI for:


Medical advice


Legal advice


Financial calculations


Security-critical code


Data about real people
Always verify:


Check sources manually


Run generated code in a sandbox


Cross-reference with official documentation


Use web search for factual claims


Part 2: Bias in AI Systems


What is AI Bias?
Definition: Systematic errors in AI outputs that reflect and amplify societal prejudices.
Critical Insight: AI doesn't create bias; it learns it from human-generated training data.


Types of Bias
1. Gender Bias


Resume screening AI: Rejected female candidates for tech roles


Google Translate: defaulted to "he" for engineer, "she" for nurse
2. Racial Bias


Facial recognition: Higher error rates for darker skin tones


Google Photos (2015 incident): Labeled Black people as "gorillas"
3. Socioeconomic Bias
Example: Lending AI trained on historical approvals learns to deny loans to certain zip codes, perpetuating discrimination.
4. Age Bias
Example: Resume screening tool penalizes graduation dates > 20 years ago.


Why Does Bias Happen?
Reason 1: Biased Training Data
If training data underrepresents certain groups, the AI learns incorrect patterns.
Reason 2: Proxy Variables
Even without using protected attributes directly, AI finds proxies (zip code for race, name for ethnicity).
Reason 3: Feedback Loops
Biased AI decisions generate biased data that trains future AI, amplifying bias over time.


Detecting Bias
Technique 1: Subgroup Analysis
Test AI performance across different groups:

test_cases = {    "male_white": ["Resume with male name, traditional university"],    "female_white": ["Resume with female name, traditional university"],    "male_minority": ["Resume with ethnic name, traditional university"],    "female_minority": ["Resume with ethnic female name, traditional university"],}for group, resumes in test_cases.items():    acceptance_rate = evaluate_ai(resumes)    print(f"{group}: {acceptance_rate}%")
Technique 2: Fairness Metrics
Demographic Parity:

P(AI predicts positive | Group A) ≈ P(AI predicts positive | Group B)
Equalized Odds:

P(AI correct | Group A, True positive) ≈ P(AI correct | Group B, True positive)
Technique 3: Counterfactual Testing
Change only protected attributes and observe output:

resume_original = {    "name": "John Smith",    "education": "MIT, BS Computer Science",    "experience": "5 years at Google"}resume_counterfactual = {    "name": "Latoya Washington",  # Changed name only    "education": "MIT, BS Computer Science",    "experience": "5 years at Google"}score_original = ai_screening(resume_original)score_counterfactual = ai_screening(resume_counterfactual)if abs(score_original - score_counterfactual) > threshold:    print("WARNING: Name bias detected")

Pillar 1: Planning
Definition: Breaking down complex goals into executable steps.

class TravelAgent:    def plan(self, goal):        """Create step-by-step plan"""        steps = [            "search_flights",            "find_hotels",            "research_attractions",            "create_itinerary",            "book_reservations"        ]        for step in steps:            self.execute_step(step)
Planning Strategies:

Pillar 2: Memory
Definition: Storing and retrieving information across interactions.
Types of Memory:
Short-Term Memory:


Current conversation context


Recent actions and their results


Temporary state
Long-Term Memory:


User preferences


Past successful strategies


Domain knowledge
Implementation:

class AgentMemory:    def __init__(self):        self.short_term = []  # Recent context        self.long_term = {}   # Persistent facts        def add_to_short_term(self, info):        self.short_term.append(info)        if len(self.short_term) > 10:            self.short_term.pop(0)        def store_long_term(self, key, value):        self.long_term[key] = value        def recall(self, key):        for item in reversed(self.short_term):            if key in item:                return item[key]        return self.long_term.get(key)

Let me verify with another source.Action: search("Jane Smith Nobel Prize 2025")Observation: Multiple sources confirm Dr. Jane Smith won.Thought: Confirmed. I can provide the answer.Final Answer: Dr. Jane Smith won the Nobel Prize in Physics in 2025.

Part 6: Error Handling in Tools

@tool("Robust API Tool")def fetch_data(endpoint: str, max_retries: int = 3) -> str:    """Fetch data from API with retry logic"""    base_url = "https://api.example.com"    for attempt in range(max_retries):        try:            response = requests.get(f"{base_url}/{endpoint}", timeout=10)            if response.status_code == 200:                return response.json()            elif response.status_code == 404:                return "Error: Endpoint not found"            elif response.status_code == 429:                import time                wait_time = 2 ** attempt  # Exponential backoff                time.sleep(wait_time)                continue            else:                return f"Error: HTTP {response.status_code}"        except requests.Timeout:            if attempt == max_retries - 1:                return "Error: Request timed out after retries"            continue        except requests.RequestException as e:            return f"Error: {str(e)}"    return "Error: Max retries exceeded"

Lecture 14: RAG: Giving your Agents "Long-Term Memory"


Introduction: The Context Window Problem
Problem:
LLMs have limited context windows:


GPT-3.5: ~4K tokens (~3,000 words)


GPT-4: ~8K-32K tokens (~25,000 words)


Claude 3: ~200K tokens (but expensive)
Challenge: How do you give an AI access to:


Your entire company wiki (millions of words)


All customer support tickets


Years of chat history
Solution: RAG (Retrieval Augmented Generation)


What is RAG?
Definition: Technique that retrieves relevant documents from a database and includes them in the LLM prompt.
Workflow:

User Question     ↓Retrieve relevant documents from database    ↓Include documents in prompt    ↓LLM generates answer using documents    ↓Response to user
Without RAG:

User: "What's our refund policy for damaged items?"LLM: "I don't have information about your specific refund policy."
With RAG:

User: "What's our refund policy for damaged items?"System: [Retrieves relevant policy document]LLM receives: User question + Policy documentLLM: "According to company policy, damaged items are eligible for a full refund within 30 days of purchase..."


Part 1: Vector Databases


Why Vectors?
Problem with Keyword Search:
Query: "How do I reset my password?"
Keyword match: "password", "reset"
Misses documents like:


"Forgot login credentials?"


"Account access recovery"


"Authentication troubleshooting"
Solution: Semantic Search
Convert text to vectors (embeddings) that capture meaning:

"How do I reset my password?"    → [0.2, 0.8, 0.1, ...]"Forgot login credentials?"      → [0.19, 0.82, 0.09, ...]  (similar!)"Best pizza restaurants"         → [0.9, 0.1, 0.7, ...]  (different)


Popular Vector Databases


ChromaDB: Easy setup, free, local - Best for development, small projects


Pinecone: Managed, scalable - Best for production, large scale


Weaviate: Open-source, flexible - Best for self-hosted production


Qdrant: Fast, Rust-based - Best for performance-critical apps

def hybrid_search(query: str, n_results: int = 5):    """Combine keyword and semantic search"""    # Semantic search    semantic_results = collection.query(        query_texts=[query],        n_results=n_results    )        # Keyword search    keywords = query.lower().split()    keyword_results = []    all_docs = collection.get()    for doc, doc_id in zip(all_docs['documents'], all_docs['ids']):        score = sum(1 for kw in keywords if kw in doc.lower())        if score > 0:            keyword_results.append((doc, score))    keyword_results.sort(key=lambda x: x[1], reverse=True)        # Combine results    combined = list(semantic_results['documents'][0])    for doc, _ in keyword_results[:n_results]:        if doc not in combined:            combined.append(doc)    return combined[:n_results]

Lecture 17: Monitoring Data Drift & Performance


Introduction: Models Degrade Over Time
Scenario:

Month 1: Model accuracy = 92% ✓Month 3: Model accuracy = 89% ⚠️Month 6: Model accuracy = 78% ✗
Why?
The world changed, but your model didn't.


What is Data Drift?
Definition: When the statistical properties of input data change over time.
Types:


Covariate Drift (Feature Drift): Input features change distribution. Example: Average customer age shifts from 35 to 45


Concept Drift: Relationship between inputs and outputs changes. Example: "Good credit score" threshold changes over time


Label Drift: Distribution of output classes changes. Example: Fraud rate increases from 1% to 5%


Part 1: Detecting Data Drift


Statistical Tests
1. Kolmogorov-Smirnov (KS) Test
Tests if two distributions are different.

from scipy import statsimport numpy as npdef detect_ks_drift(reference_data, current_data, threshold=0.05):    """Detect drift using KS test"""    statistic, p_value = stats.ks_2samp(reference_data, current_data)    is_drift = p_value < threshold    return {        "is_drift": is_drift,        "p_value": p_value,        "statistic": statistic    }# Exampletrain_ages = np.random.normal(35, 10, 1000)  # Mean: 35, Std: 10prod_ages = np.random.normal(45, 10, 1000)   # Mean: 45, Std: 10 (drift!)result = detect_ks_drift(train_ages, prod_ages)print(f"Drift detected: {result['is_drift']}")print(f"P-value: {result['p_value']:.4f}")# Drift detected: True# P-value: 0.0001
2. Population Stability Index (PSI)
Measures how much a distribution has shifted.

def calculate_psi(expected, actual, bins=10):    """Calculate Population Stability Index        PSI < 0.1: No significant change    0.1 <= PSI < 0.25: Moderate change    PSI >= 0.25: Significant drift    """    import numpy as np        breakpoints = np.percentile(expected, np.linspace(0, 100, bins + 1))        expected_percents = np.histogram(expected, bins=breakpoints)[0] / len(expected)    actual_percents = np.histogram(actual, bins=breakpoints)[0] / len(actual)        expected_percents = expected_percents + 0.0001    actual_percents = actual_percents + 0.0001        psi_values = (actual_percents - expected_percents) * np.log(actual_percents / expected_percents)    psi = np.sum(psi_values)    return psi# Exampletrain_data = np.random.normal(100, 15, 10000)prod_data = np.random.normal(110, 15, 10000)psi = calculate_psi(train_data, prod_data)print(f"PSI: {psi:.4f}")if psi < 0.1:    print("No significant change")elif psi < 0.25:    print("Moderate drift - monitor closely")else:    print("Significant drift - retrain model")

import pandas as pdfrom datetime import datetime, timedeltafrom sklearn.metrics import accuracy_score, precision_score, recall_score, f1_scoreclass ModelMonitor:    def __init__(self):        self.predictions_log = []        def log_prediction(self, features, prediction, actual=None, timestamp=None):        """Log model prediction"""        if timestamp is None:            timestamp = datetime.now()                log_entry = {            'timestamp': timestamp,            'features': features,            'prediction': prediction,            'actual': actual        }        self.predictions_log.append(log_entry)        def calculate_metrics(self, time_window_days=7):        """Calculate performance metrics for recent predictions"""        cutoff_time = datetime.now() - timedelta(days=time_window_days)        recent_logs = [            log for log in self.predictions_log             if log['timestamp'] >= cutoff_time and log['actual'] is not None        ]                if not recent_logs:            return None                y_true = [log['actual'] for log in recent_logs]        y_pred = [log['prediction'] for log in recent_logs]                return {            'accuracy': accuracy_score(y_true, y_pred),            'precision': precision_score(y_true, y_pred, average='weighted'),            'recall': recall_score(y_true, y_pred, average='weighted'),            'f1': f1_score(y_true, y_pred, average='weighted'),            'num_predictions': len(recent_logs)        }        def detect_performance_degradation(self, baseline_accuracy=0.90, threshold=0.05):        """Check if performance has degraded significantly"""        metrics = self.calculate_metrics(time_window_days=7)        if metrics is None:            return False        current_accuracy = metrics['accuracy']        degradation = baseline_accuracy - current_accuracy        return degradation > threshold

import pandas as pdfrom fairlearn.metrics import MetricFrame, demographic_parity_differencefrom sklearn.metrics import accuracy_score, precision_scoreclass BiasAuditor:    def __init__(self, model, X_test, y_test, sensitive_features):        self.model = model        self.X_test = X_test        self.y_test = y_test        self.sensitive_features = sensitive_features        self.y_pred = model.predict(X_test)        def calculate_overall_metrics(self):        """Calculate overall performance"""        return {            'accuracy': accuracy_score(self.y_test, self.y_pred),            'precision': precision_score(self.y_test, self.y_pred, average='weighted')        }        def calculate_group_metrics(self):        """Calculate metrics by sensitive group"""        metric_frame = MetricFrame(            metrics={'accuracy': accuracy_score, 'precision': precision_score},            y_true=self.y_test,            y_pred=self.y_pred,            sensitive_features=self.sensitive_features        )        return metric_frame.by_group        def check_fairness(self):        """Check fairness constraints"""        dp_diff = demographic_parity_difference(            y_true=None,            y_pred=self.y_pred,            sensitive_features=self.sensitive_features        )        return {            'demographic_parity_diff': dp_diff,            'is_fair': abs(dp_diff) < 0.1  # Threshold        }        def generate_report(self):        """Generate complete bias audit report"""        report = {            'overall_metrics': self.calculate_overall_metrics(),            'group_metrics': self.calculate_group_metrics().to_dict(),            'fairness_check': self.check_fairness()        }        return report# Usageauditor = BiasAuditor(    model=trained_model,    X_test=X_test,    y_test=y_test,    sensitive_features=df_test['gender'])report = auditor.generate_report()print(json.dumps(report, indent=2))

class FraudDetectionSystem:    def __init__(self, model, rules):        self.model = model        self.rules = rules        def score_transaction(self, transaction):        """Score transaction for fraud risk"""        # 1. Extract features        features = self.extract_features(transaction)                # 2. Apply rules (fast path for obvious cases)        rule_decision = self.apply_rules(transaction)        if rule_decision:            return {                'risk_score': 100 if rule_decision == 'BLOCK' else 0,                'decision': rule_decision,                'explanation': 'Rule-based decision',                'rule_triggered': True            }                # 3. ML model scoring        risk_score = self.model.predict_proba([features])[0][1] * 100                # 4. Make decision        if risk_score > 80:            decision = 'BLOCK'        elif risk_score > 50:            decision = 'REVIEW'        else:            decision = 'APPROVE'                explanation = self.explain(features, risk_score)                return {            'risk_score': risk_score,            'decision': decision,            'explanation': explanation,            'rule_triggered': False        }        def apply_rules(self, transaction):        """Apply hard business rules"""        if transaction['amount'] > 10000 and transaction['merchant'] == 'NEW':            return 'REVIEW'        if transaction['is_foreign'] and transaction['card_not_present']:            return 'REVIEW'        if transaction['amount'] > 50000:            return 'BLOCK'        return None

from presidio_analyzer import AnalyzerEnginefrom presidio_anonymizer import AnonymizerEngineclass HIPAACompliantDataProcessor:    def __init__(self):        self.analyzer = AnalyzerEngine()        self.anonymizer = AnonymizerEngine()        def de_identify_text(self, text):        """Remove PHI (Protected Health Information) from text                PHI includes:        - Names, Dates (except year), Phone numbers        - Email addresses, Medical record numbers, IP addresses        """        results = self.analyzer.analyze(            text=text,            entities=[                "PERSON", "EMAIL_ADDRESS", "PHONE_NUMBER",                "MEDICAL_LICENSE", "US_SSN"            ],            language='en'        )                anonymized_text = self.anonymizer.anonymize(            text=text,            analyzer_results=results        )        return anonymized_text.text        def process_medical_record(self, record):        """Process medical record while maintaining HIPAA compliance"""        processed = {}                if 'notes' in record:            processed['notes'] = self.de_identify_text(record['notes'])                # Remove direct identifiers        processed['patient_id'] = self.hash_identifier(record['patient_id'])                # Keep clinical data        processed['diagnosis_codes'] = record.get('diagnosis_codes', [])        processed['lab_results'] = record.get('lab_results', [])                # Generalize dates (keep only year)        if 'admission_date' in record:            processed['admission_year'] = record['admission_date'][:4]                return processed        def hash_identifier(self, identifier):        """One-way hash for patient ID"""        import hashlib        return hashlib.sha256(identifier.encode()).hexdigest()[:16]

2. A/B Testing in Production

class ABTestFramework:    def __init__(self, model_a, model_b, split_ratio=0.5):        self.model_a = model_a  # Current production model        self.model_b = model_b  # New challenger model        self.split_ratio = split_ratio        self.results_a = []        self.results_b = []        def predict(self, features, user_id):        """Route prediction to A or B based on user_id"""        import hashlib                hash_value = int(hashlib.md5(user_id.encode()).hexdigest(), 16)        use_model_b = (hash_value % 100) < (self.split_ratio * 100)                if use_model_b:            prediction = self.model_b.predict([features])[0]            self.results_b.append({'user_id': user_id, 'prediction': prediction})            model_used = 'B'        else:            prediction = self.model_a.predict([features])[0]            self.results_a.append({'user_id': user_id, 'prediction': prediction})            model_used = 'A'                return {            'prediction': prediction,            'model_used': model_used,            'ab_test_version': 'v1.2'        }        def evaluate_test(self, feedback_data):        """Evaluate A/B test results"""        from sklearn.metrics import accuracy_score        from scipy import stats                y_true_a = [feedback_data[r['user_id']] for r in self.results_a if r['user_id'] in feedback_data]        y_pred_a = [r['prediction'] for r in self.results_a if r['user_id'] in feedback_data]                y_true_b = [feedback_data[r['user_id']] for r in self.results_b if r['user_id'] in feedback_data]        y_pred_b = [r['prediction'] for r in self.results_b if r['user_id'] in feedback_data]                acc_a = accuracy_score(y_true_a, y_pred_a)        acc_b = accuracy_score(y_true_b, y_pred_b)                _, p_value = stats.ttest_ind(y_true_a, y_true_b)                return {            'model_a_accuracy': acc_a,            'model_b_accuracy': acc_b,            'difference': acc_b - acc_a,            'p_value': p_value,            'is_significant': p_value < 0.05,            'recommendation': 'Adopt Model B' if (acc_b > acc_a and p_value < 0.05) else 'Keep Model A'        }

Glossary

AI Agent: Autonomous system that can plan, use tools, and work toward goals

Artifact: File created as output (model, report, visualization)

Bias: Systematic error in predictions, often along demographic lines

Container: Isolated runtime environment with code and dependencies

Data Drift: Change in input data distribution over time

Deployment: Making a model available for production use

Docker: Platform for containerizing applications

Embedding: Numerical representation of text, images, etc.

Fine-tuning: Adapting pre-trained model to specific task

GCC: Global Capability Center (offshore development center)

Gradient Boosting: Ensemble method building trees sequentially

Hyperparameter: Configuration setting for model training

Inference: Making predictions with a trained model

LLM: Large Language Model (e.g., GPT-4, Claude)

MLOps: Practices for deploying and maintaining ML systems

Model Card: Documentation of model purpose, performance, limitations

Overfitting: Model performs well on training data, poorly on new data

Pipeline: Series of data processing steps

RAG: Retrieval Augmented Generation (combining search + LLM)

SHAP: Method for explaining model predictions

Vector Database: Database optimized for similarity search
